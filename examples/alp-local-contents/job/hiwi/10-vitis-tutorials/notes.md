# Vitis Tutorials Notes

## practical reading strategy

Read this repository like a map, not like a textbook.

The useful question is:

```text
If someone gives me an HLS lab tomorrow, do I understand where the source,
testbench, synthesis scripts, target part, reports, and exported IP live?
```

## keep in mind

Tool versions matter a lot in AMD/Xilinx workflows. Do not spend hours debugging
a local setup before supervisors confirm which version the lab expects.

## useful one-liner for the meeting

I looked at the broad Vitis tutorials as orientation, but I focused my preparation
on the HLS flow because that seems closest to the HiWi task: C/C++ top functions,
testbenches, synthesis, reports, co-simulation, and IP export.
