# alpnest

status: active
role: sanity-check point of contact
current phase: cockpit baseline finished, project observability next

## purpose

alpnest is the local cockpit before work starts.

It should become the first place to check:

- today's tasks
- active school/course work
- project state
- job/HiWi context
- important mail
- upcoming calendar events
- local git/repository health

The goal is not to replace the terminal, editor, mailbox, or calendar. The goal is to give one calm local surface that tells Alp what needs attention before opening the rest of the toolchain.

## active milestones

### ms1: local git tracker

priority: highest
status: next

Build a local repository tracker for `~/Documents/GitHub`.

The tracker should discover directories containing `.git`, then generate project-level git summaries. alpnest should show dirty repositories, current branch, recent commits, remote information, and unpushed/uncommitted work.

Expected outputs:

- `data/projects/<repo>/git.md`
- generated project overview summary
- dirty repository visibility in the projects panel

### ms2: readability and theme pass

priority: high
status: planned

Make the TUI easier to read for daily use. Improve contrast, colors, selected-row styling, labels, borders, and compact/fullscreen behavior.

The target is a mature terminal cockpit, not a toy demo.

### ms3: calendar tracker and push notifications

priority: high
status: planned after git tracker and theme pass

Start with read-only calendar sync. Generate today's calendar snapshot, upcoming events, meeting reminders, and deadline context. Notifications should be meaningful and rare.

Write access to calendar should come later and only with explicit safety checks.

### ms4: project milestone drill-down

priority: medium
status: planned

Project overview items should open milestone detail files. Milestones should be arbitrary markdown files under each project folder and should be editable by Alp or generated by an LLM-assisted planning step.

### ms5: daemon hardening

priority: medium
status: planned

Background sync should become reliable: lockfiles, logs, stale-state handling, notification deduplication, and safer launchd usage.
